Sprites

MacScheme’s sprite system sits alongside the framebuffer API. Sprite definitions hold pixel data and palettes; sprite instances hold position, visibility, rotation, scale, animation, and effects.

Two good ways to author sprites

  • Use sprite-from-rows! when a sprite is easiest to sketch as rows of palette indices. Width comes from the first row, height from the number of rows.
  • Use with-sprite-canvas with drawing primitives when you want circles, boxes, triangles, lines, or procedurally generated shapes.

Typical flow

(gfx-screen 320 240 1)
(gfx-sprite-def 0 16 16)
; populate pixels or rows here
(gfx-sprite-commit 0)
(gfx-sprite 0 0 120 80)
(gfx-sprite-sync)
(gfx-flip)

Useful commands

  • gfx-sprite-def, gfx-sprite-data, and gfx-sprite-commit define sprite pixel data.
  • gfx-sprite creates or places an instance on screen.
  • gfx-sprite-pos, gfx-sprite-move, gfx-sprite-rot, and gfx-sprite-scale control transforms.
  • gfx-sprite-frame and gfx-sprite-animate drive sprite sheet animation.
  • gfx-sprite-sync flushes the latest sprite state before presenting.

Examples in MacScheme

  • examples/sprite_demo.ss for row-authored sprites
  • examples/sprite_shapes_demo.ss for primitive-authored shape sprites
  • examples/sprite_mixed_demo.ss for a mixed scene using both techniques
